Using Existing Samples

If you download and imports any existing samples form the net, the default settings should suffice. Things to watch up for are:

·         Output folder.
Assure that the Update Bundle-ClassPath on Activate/Deactivate is switched on.

·         Lazy Activation Policy

If the activation policy is set to lazy, the bundle will enter state “LAZY” (STARTING). To start the bundle (state ACTIVE) you have to change the policy to eager either from the pop-up menu in the List or Details View of the bundle, in the overview tab in the manifest editor  or directly in the manifest file.

·         Build Automatically
If Build Automatically is switched off the sample bundle will not get updated before it is built. If on, the bundle is updated when you save any file in the project.
