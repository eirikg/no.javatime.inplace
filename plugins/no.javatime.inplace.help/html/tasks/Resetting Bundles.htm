How To

To reset the entire workspace region, select Reset from the Bundle main menu. To reset an individual bundle and all its dependent bundles select Reset from the context menu in the Package Explorer, Bundle Detail Page View or the Bundle List Page View.

Reset

Reset first stops, uninstalls and refreshes the bundles before they are installed, resolved and started again. For bundles in state resolve they return to the same state after being Reset. Deactivated bundle projects are uninstalled, refreshed and then installed again. Performing a Project | Clean on all projects followed by a Bundle | Reset of the workspace results in both a clean state for both projects and bundles.

Reset the Workspace Region

Resetting workspace bundles is a way to get a clean workspace state where the latest build is included and all revisions are released and new revisions created. After reset, all workspace bundles will only have one revision and be based on the most recent build. If there are any bundles with errors when trying to reset the workspace, the operation is cancelled.

Resetting Individual Bundles

When selecting a bundle to reset from the Bundle View, one has to consider both the closure that are required to uninstall the bundles and from this closure add the bundles that are required to install and resolve the uninstalled bundles again. Thus, reset is a two-step process applying the Providing and Requiring closure as described in Setting Bundle Options. Before reset, all bundles with errors and their Providing and Requiring closures are removed from the set of bundles to reset.
